1. 인증 및 보안 (Authentication & Security)
 비로그인 글쓰기 시도 차단:
로그아웃 상태에서 

createPost
 액션이 호출되었을 때, "User not authenticated" 에러가 발생하는지 확인 (네트워크 탭이나 서버 로그 확인).
 권한 없는 수정/삭제 차단 (RLS):
내 글이 아닌 다른 사람의 글(

id
)에 대해 

updatePost
나 

deletePost
를 호출했을 때, DB 수준에서 차단되어 에러가 반환되거나 변경사항이 없어야 함.
 작성자 이름 위변조 방지:
글 작성 시 author_name이 클라이언트에서 보내는 값이 아니라, 서버 세션의 사용자 정보에서 가져와지는지 코드 확인.
2. 데이터 유효성 검사 (Data Validation)
 필수값 누락 처리:
title이나 content가 빈 문자열("")로 넘어왔을 때 서버가 어떻게 반응하는지. (현재 코드상 타입 캐스팅만 되어 있어, DB 제약조건 에러가 발생할 가능성 확인)
 잘못된 카테고리:
정의되지 않은 카테고리 문자열을 보냈을 때 저장이 되는지, 아니면 무시되는지 확인.
3. 에러 핸들링 (Error Handling)
 DB 연결 실패 시:
Supabase 연결이 끊기거나 타임아웃 발생 시, 사용자에게 "Failed to create post" 같은 에러가 적절히(Toast 메시지 등) 전달되는지.
 존재하지 않는 게시글 조회:
수정 페이지(/edit) 등에서 존재하지 않는 

id
로 

getPostById
를 호출했을 때 null이 반환되고, 프론트엔드에서 404 페이지로 잘 이동하는지.
4. 캐싱 및 최신 데이터 반영 (Caching & Revalidation)
 글 작성 후 목록 갱신:

createPost
 후 /community와 /community/[category] 페이지가 즉시 갱신(revalidatePath)되어, 새로고침 없이도 내 글이 보이는지.
 글 수정 후 상세 페이지 갱신:

updatePost
 후 상세 페이지로 리다이렉트되었을 때, 수정된 내용이 바로 반영되어 있는지(캐시가 남아있지 않은지).
5. 성능 및 최적화
 불필요한 데이터 요청:

getPostsByCategory
에서 select("*")를 사용 중인데, 목록 뷰에서 content 전체를 가져오는 것이 너무 무겁지 않은지(필요하다면 필드 제한 고려).
